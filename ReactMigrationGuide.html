Create Your React App

💡 npm · project scaffolding · entry point ⏱ 20–30 min

The first step is to create a brand-new React project using Create React App (CRA). CRA is an official tool from the React team that downloads and configures everything automatically — React, a local dev server, and hot-reloading.

Scaffold the Project

Open a terminal in VS Code (Terminal → New Terminal) and run:

bash
# Create a new React project using Create React App
npx create-react-app react-blog

# Navigate into the project folder
cd react-blog

# Start the development server
npm start

After npm start, the browser opens at http://localhost:3000 showing the default React starter page.

⚠️
Note: This may take a few minutes
The first run of npx create-react-app downloads several packages. Ensure students have a stable internet connection. The terminal prints "Happy hacking!" when ready.

Understand the Generated Files

FilePurpose
public/index.htmlThe ONLY HTML file. React mounts itself here. Students rarely edit this.
src/index.jsThe JavaScript entry point. Connects React to the HTML page.
src/App.jsThe root React component. This is where we start building.
src/App.cssStyles for App.js. We will replace this with our own styles.
package.jsonLists all dependencies and available scripts (start, build, test).

Clean Up the Starter Template

Replace src/App.js with a clean slate:

Convert HTML Pages to Components

💡 JSX · React components · HTML→JSX syntax ⏱ 45–60 min

A React component is a JavaScript function that returns HTML-like code called JSX. In this step, students convert each of their 5 HTML pages into individual React component files.

JSX vs HTML — Key Differences

❌ HTML (Old)✅ JSX (React)
<div class="nav"><div className="nav">
<img src="..." /><img src="..." /> (same!)
<label for="email"><label htmlFor="email">
onclick="doThing()"onClick={"{doThing}"}
style="color: red;"style={{ color: "red" }}
<!-- comment -->{"{/* comment */}"}
📝
Key Rule
In JSX, HTML attributes that conflict with JS reserved words are renamed. The most common: class → className, for → htmlFor, onclick → onClick.

03

Add Routing Between Pages

💡 React Router · BrowserRouter · Routes · Link · useNavigate ⏱ 40–50 min

In vanilla HTML, navigation loads a new HTML file. In React there is only ONE HTML file — React Router intercepts navigation and swaps components in and out without reloading the page.

Install React Router

bash
npm install react-router-dom

Set Up Routes in App.js

jsx · src/App.js
// src/App.js
import { BrowserRouter, Routes, Route } from 'react-router-dom';
import SplashPage from './pages/SplashPage';
import HomePage from './pages/HomePage';
import AboutPage from './pages/AboutPage';
import ContactPage from './pages/ContactPage';
import RegisterPage from './pages/RegisterPage';

function App() {
  return (
    <BrowserRouter>
      <Routes>
        <Route path='/'         element={<SplashPage />} />
        <Route path='/home'     element={<HomePage />} />
        <Route path='/about'    element={<AboutPage />} />
        <Route path='/contact'  element={<ContactPage />} />
        <Route path='/register' element={<RegisterPage />} />
      </Routes>
    </BrowserRouter>
  );
}
export default App;

Add Navigation Links

Replace all <a href="home.html"> links with React Router's <Link to="/home">:

jsx · SplashPage.js
import { Link } from 'react-router-dom';

function SplashPage() {
  return (
    <div className='splash'>
      <h1>Welcome to My Blog</h1>
      <p>Stories worth reading.</p>
      {/* Use <Link> instead of <a href> */}
      <Link to='/home' className='btn'>
        Enter
      </Link>
    </div>
  );
}
⚠️
Why not use <a href>?
A regular <a href> causes a full page reload, losing React's state and defeating the purpose of a Single Page Application. <Link> handles navigation without reloading.

Create the Shared Navigation Component

jsx · src/components/Nav.js
import { Link, useLocation } from 'react-router-dom';

function Nav() {
  const location = useLocation();
  const links = [
    { path: '/home',     label: 'Home'    },
    { path: '/about',    label: 'About'   },
    { path: '/contact',  label: 'Contact' },
    { path: '/register', label: 'Join'    },
  ];
  return (
    <nav className='navbar'>
      <Link to='/' className='logo'>MyBlog</Link>
      <ul>
        {links.map(link => (
          <li key={link.path}>
            <Link
              to={link.path}
              className={location.pathname === link.path ? 'active' : ''}
            >
              {link.label}
            </Link>
          </li>
        ))}
      </ul>
    </nav>
  );
}
export default Nav;
✏️
Student Discussion
Why do we put Nav in a separate file? What happens if we copy Nav code into every page? This leads to discussing the DRY principle — Don't Repeat Yourself.

04

Add State with useState

💡 React hooks · useState · form inputs · conditional rendering ⏱ 45–60 min

In vanilla JS you'd use document.getElementById('input').value to read form values. In React, we use the useState hook to track values and keep the UI in sync automatically.

The useState Hook Explained

Start with a simple counter before jumping into forms:

jsx · Counter example
import { useState } from 'react';

function Counter() {
  // ↓ current value   ↓ function to change it
  const [count, setCount] = useState(0); // ← starting value

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>
        Add 1
      </button>
    </div>
  );
}
⚠️
Key Rule
Never modify state directly. Always use the setter function (e.g., setCount). React only re-renders the component when you use the setter.

Controlled Form Inputs

jsx · src/pages/ContactPage.js
import { useState } from 'react';

function ContactPage() {
  const [formData, setFormData] = useState({
    name: '', email: '', message: '',
  });

  // Single handler for ALL fields
  const handleChange = (e) => {
    setFormData({
      ...formData,             // keep existing values
      [e.target.name]: e.target.value, // update changed field
    });
  };

  const handleSubmit = () => {
    console.log('Form submitted:', formData);
    alert(`Thanks, ${formData.name}! Message received.`);
  };

  return (
    <div className='contact-page'>
      <h1>Contact Us</h1>
      <input type='text'  name='name'    value={formData.name}
             onChange={handleChange} placeholder='Your Name'  />
      <input type='email' name='email'   value={formData.email}
             onChange={handleChange} placeholder='Your Email' />
      <textarea name='message' value={formData.message}
                onChange={handleChange} placeholder='Your message...' />
      <button onClick={handleSubmit}>Send Message</button>
    </div>
  );
}
export default ContactPage;

Conditional Rendering (Success State)

jsx · success state pattern
const [submitted, setSubmitted] = useState(false);
const handleSubmit = () => { setSubmitted(true); };

// In the JSX:
return (
  <div>
    {submitted ? (
      <p>Thank you! We will get back to you soon.</p>

      <form>...your form here...</form>

05

Migrate CSS Styling

💡 CSS Modules · global styles · inline styles in React ⏱ 30–40 min

Students' existing CSS can be reused in React, but there are better, more organized ways. This step shows three approaches — from simplest to most professional.

Option A — Global Stylesheet (Easiest)

Simply import the existing CSS file into index.js. All classes become available globally:

jsx · src/index.js
import './index.css'; // or import './style.css'
ℹ️
Best For
Quick migrations where students want to reuse their existing CSS without changes. Good starting point.

Option B — CSS Modules (Recommended)

CSS Modules give each component its own scoped CSS file. Classes from one file won't accidentally affect other components.

css · src/pages/HomePage.module.css
.container { max-width: 1100px; margin: 0 auto; padding: 2rem; }
.title     { font-size: 2.5rem; color: #1E4D8C; }
.postCard  { border: 1px solid #ddd; padding: 1.5rem; border-radius: 8px; }
jsx · src/pages/HomePage.js
import styles from './HomePage.module.css';

function HomePage() {
  return (
    <div className={styles.container}>
      <h1 className={styles.title}>Latest Posts</h1>
      <div className={styles.postCard}>Post 1</div>
    </div>
  );
}
ℹ️
Why Use CSS Modules?
In large projects, two developers might use the same class name (e.g., both use .title). CSS Modules automatically rename classes to be unique so styles from one component never bleed into another.

Option C — Inline Styles (For Dynamic Styles)

jsx · dynamic inline styles
// style takes a JS object — property names use camelCase
function PostCard({ isActive }) {
  return (
    <div style={{
      border: '1px solid #ddd',
      padding: '1.5rem',
      backgroundColor: isActive ? '#E8F4FD' : 'white',
      transition: 'background-color 0.3s',
    }}>
      Post content here
    </div>
  );
}

Rendering Lists & Blog Posts

💡 .map() · props · component reuse · key prop ⏱ 40–50 min

Instead of writing 10 blog post cards manually in HTML, write one PostCard component and let React repeat it for each item in your data array.

Create a Data Array

js · src/data/posts.js
const posts = [
  {
    id: 1,
    title: 'Getting Started with React',
    excerpt: 'React is a JavaScript library for building UIs...',
    date: 'Feb 28, 2025',
    tag: 'Technology',
    readTime: '5 min',
  },
  {
    id: 2,
    title: 'Why CSS Still Matters',
    excerpt: 'In a world of utility frameworks, understanding core CSS...',
    date: 'Feb 15, 2025',
    tag: 'Design',
    readTime: '4 min',
  },
  // Add more posts here...
];

export default posts;

Create a Reusable PostCard Component

jsx · src/components/PostCard.js
function PostCard({ title, excerpt, date, tag, readTime }) {
  return (
    <div className='post-card'>
      <span className='tag'>{tag}</span>
      <h2>{title}</h2>
      <p>{excerpt}</p>
      <div className='meta'>
        <span>{date}</span>
        <span>{readTime} read</span>
      </div>
    </div>
  );
}
export default PostCard;

Render the List in HomePage

jsx · src/pages/HomePage.js
import posts from '../data/posts';
import PostCard from '../components/PostCard';

function HomePage() {
  return (
    <div className='home-page'>
      <h1>Latest Posts</h1>
      <div className='posts-grid'>
        {/* .map() loops through array and returns JSX for each item */}
        {posts.map(post => (
          <PostCard
            key={post.id}        {/* REQUIRED: unique identifier */}
            title={post.title}
            excerpt={post.excerpt}
            date={post.date}
            tag={post.tag}
            readTime={post.readTime}
          />
        ))}
      </div>
    </div>
  );
}
export default HomePage;
⚠️
The key Prop is Required
When rendering a list with .map(), every item needs a unique key prop. Use the item's unique ID (key={"{post.id}"}), never the index (key={"{i}"}) if the list can change order.

Concept Cheat Sheet

Component

≈ Separate .html file

One per page or reusable UI block

JSX

≈ HTML inside .html files

Describing what the UI looks like

useState

≈ querySelector + variables

Any value that can change over time

Props

≈ URL params / hidden inputs

Sending data from parent to child

.map()

≈ for loop + innerHTML

Rendering a list of items from an array

React Router

≈ Links to separate HTML files

Navigating without reload

CSS Modules

≈ CSS files linked in <head>

Scoped styles per component

Common Error Messages & Fixes

Error MessageMost Likely Fix
Each child in a list should have a unique "key" propAdd key={"{item.id}"} to your .map() return element
Cannot read properties of undefinedVariable is undefined — check your import path or state initialization
JSX expressions must have one parent elementWrap multiple elements in a <div> or empty <></> fragment
Module not found: can't resolve './MyComponent'Check file name and path — React imports are case-sensitive
React Hook "useState" cannot be called inside a loopMove useState calls to the top of the component, never inside loops or ifs
