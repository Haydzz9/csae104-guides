Overview & Prerequisites

What you need before you start

Before deploying, make sure your blog project is working locally. You need accounts on three free platforms:

⎇ GitHub
Hosts your source code. Render and Vercel both connect directly to your GitHub repos to auto-deploy on every push.
github.com — Free
⚡ Render
Hosts your Express API and PostgreSQL database. Free tier includes 750 hrs/month of hosting.
render.com — Free Tier
▲ Vercel
Hosts your React frontend. Deploys automatically from GitHub. Gives you a custom .vercel.app URL.
vercel.com — Free Hobby Plan
🛠 VS Code + Git
Make sure Git is installed on your computer. Check with git --version in your terminal.
git-scm.com — Free

Prerequisite Checklist

  • Blog site works on localhost:3000 (React) and localhost:5000 (Express)
  • Account created at github.com
  • Account created at render.com
  • Account created at vercel.com
  • Git installed — run git --version to confirm
  • VS Code integrated terminal is working (Ctrl+`)
💡
Sign up with your GitHub account on both Render and Vercel — this makes linking your repos much easier and avoids extra steps.

Initialize Git — Backend (blog-server)

Set up .gitignore, commit, and link to GitHub

Open VS Code and navigate to your blog-server folder in the terminal.

First, create a .gitignore file

This prevents sensitive files (like your .env) and heavy folders (like node_modules) from being uploaded to GitHub.

🔒 blog-server/.gitignore
# Dependencies — never commit these (huge folder)
node_modules/

# Environment variables — NEVER commit (contains passwords!)
.env

# OS junk
.DS_Store
Thumbs.db

# Logs
*.log
npm-debug.log*
🔴
Never commit your .env file! It contains your database password. The .gitignore above prevents this — always double-check before pushing.

Initialize and push to GitHub

Copy your GitHub repo URL — it looks like https://github.com/YOUR-USERNAME/blog-server.git

📟 Terminal — inside blog-server/ bash
# Step 1 — Initialize a new Git repository
git init

# Step 2 — Stage all files (excluding .gitignore'd ones)
git add .

# Step 3 — Make your first commit
git commit -m "Initial commit: Express blog API"

# Step 4 — Rename the default branch to 'main'
git branch -M main

# Step 5 — Link to your GitHub repo (replace with YOUR URL)
git remote add origin https://github.com/YOUR-USERNAME/blog-server.git

# Step 6 — Push to GitHub
git push -u origin main

Verify it worked

Visit github.com/YOUR-USERNAME/blog-server — you should see your files listed (but no .env or node_modules).

📁 blog-server ← on GitHub
  📄 server.js ✓ uploaded
  📄 db.js ✓ uploaded
  📁 routes/ ✓ uploaded
    📄 posts.js ✓ uploaded
  📄 package.json ✓ uploaded
  📄 .gitignore ✓ uploaded
  📄 .env ← NOT uploaded (hidden by .gitignore) ✓
  📁 node_modules/ ← NOT uploaded ✓

Making Future Updates (Git Push Workflow)

The 3-command workflow you'll use every time you update your code

Every time you make changes to your code and want to update GitHub (and auto-deploy), follow this workflow:

📟 The standard Git push workflow bash
# 1. Check what files changed
git status

# 2. Stage your changes (. means all files)
git add .

# 3. Write a descriptive commit message
git commit -m "Fix: add category filter to posts API"

# 4. Push to GitHub (Render & Vercel auto-redeploy!)
git push

Essential Git Commands Reference

📖 Git Cheatsheet
git status          # see changed/staged files
git add .            # stage all changes
git add <filename>   # stage one file
git commit -m "msg"  # save a snapshot
git push             # upload to GitHub
git pull             # download latest changes
git log --oneline    # see commit history
git diff             # see what changed (not staged)
git restore .        # undo unstaged changes
🌟
Good commit messages matter! Write messages like "Add: post edit functionality" or "Fix: 404 error on post delete" — not just "changes". Your future self will thank you.

Provision PostgreSQL on Render

Create a free cloud database and connect it to your API

Your local PostgreSQL won't be accessible from the internet. Create a new PostgreSQL database on Render for free.

Create a PostgreSQL instance

1
In Render dashboard: "New +" → "PostgreSQL"
2
Name: blog-db · Region: Singapore · Plan: Free
3
Click "Create Database"
4
Wait for status to show ● Available
5
Copy the Connection Details (shown on the DB info page)

Add DB credentials to your Render Web Service

Go to your blog-server Web Service → Environment tab → Add Environment Variables:

🔑 Render — Environment Variables (from your DB info page)
DB_HOST     = dpg-xxxxxx.singapore-postgres.render.com
DB_PORT     = 5432
DB_NAME     = blog_db
DB_USER     = blog_db_user
DB_PASSWORD = your-render-generated-password
PORT        = 5000
💡
Internal vs External URL: Use the Internal Database URL for DB_HOST (faster, no data transfer cost). Render auto-connects services within the same region.

Create your tables in the cloud database

Use the Render database's PSQL Command (shown in the DB info page) to run your schema. Or install the VS Code PostgreSQL extension and connect with the External Connection String.

📟 Run schema via PSQL (copy the command from Render) bash
# Render provides a command like this — copy & paste it from your DB page:
PGPASSWORD=your-password psql -h dpg-xxx.singapore-postgres.render.com \
  -U blog_db_user -d blog_db -f schema.sql

# OR paste this SQL manually into the Render "PSQL Console":
CREATE TABLE posts (
  id         SERIAL PRIMARY KEY,
  title      VARCHAR(255) NOT NULL,
  content    TEXT NOT NULL,
  author     VARCHAR(100) NOT NULL,
  category   VARCHAR(60) DEFAULT 'General',
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

Update Environment Variables

Make sure all URLs point to production, not localhost

Now that both services are live, double-check that every URL is pointing to the deployed services — not localhost.

Summary of all environment variables

📋 Complete Environment Variable Reference
── RENDER (blog-server Web Service — Environment tab) ─────────
DB_HOST     = dpg-xxxxxx.singapore-postgres.render.com
DB_PORT     = 5432
DB_NAME     = blog_db
DB_USER     = blog_db_user
DB_PASSWORD = <from Render DB page>
PORT        = 5000

── VERCEL (blog-client — Settings → Environment Variables) ────
REACT_APP_API_URL = https://blog-server-xxxx.onrender.com/api

Update CORS in your Express server

Your Express server needs to allow requests from your Vercel URL. Update server.js:

⚡ blog-server/server.js — update CORS JS
const cors = require('cors')

// Allow requests from both localhost (dev) and Vercel (production)
app.use(cors({
  origin: [
    'http://localhost:3000',
    'https://blog-client-xxxx.vercel.app',   // ← your Vercel URL
    'https://your-custom-domain.com'          // ← optional custom domain
  ],
  methods: ['GET', 'POST', 'PUT', 'DELETE'],
  credentials: true
}))

Then push the CORS change to GitHub — Render will auto-redeploy:

📟 Terminal — inside blog-server/
git add server.js
git commit -m "Update CORS to allow Vercel frontend"
git push

Test Your Live Site!

Verify everything is working end-to-end in production

Open your Vercel URL in a browser and run through this checklist:

  • Homepage loads and shows blog posts from the database
  • Category filter buttons work (React, CSS, etc.)
  • Clicking a post opens the detail page with full content
  • "+ New Post" form submits and creates a post in the cloud database
  • Edit post updates the content and redirects back
  • Delete post removes it and redirects to homepage
  • Open your API URL directly: https://blog-server-xxxx.onrender.com/api/posts
  • Share the Vercel URL with a classmate — it works for them too!

Common Issues & Fixes

🔧 Troubleshooting Guide
❌ "Cannot GET /api/posts" — 404 error
   → Check Render logs: Dashboard → blog-server → Logs tab
   → Confirm Start Command is: node server.js

❌ CORS error in browser console
   → Add your Vercel URL to the CORS origin array in server.js
   → Push the change and wait for Render to redeploy

❌ "DB connection error: password authentication failed"
   → Recheck DB_PASSWORD env var in Render — no spaces around "="
   → Copy-paste directly from Render DB info page

❌ Posts not loading on Vercel (blank page or error)
   → Check REACT_APP_API_URL in Vercel Environment Variables
   → Redeploy: Vercel Dashboard → Deployments → Redeploy

❌ First API request takes ~30 seconds
   → Normal on free tier — Render spins down idle services
   → Second request onwards is fast

✓ To see real-time server logs:
   Render Dashboard → blog-server → Logs

✓ To redeploy manually:
   Render → blog-server → Manual Deploy → Deploy latest commit
